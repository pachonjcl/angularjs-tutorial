Framework Estructural

AngularJS MVC

MVC es un patrón de diseño de software para crear interfaces de usuario. MVC divide una aplicación en tres componentes interconectados para separar la representación interna de la información de la forma que es presentada al usuario. En aplicaciones web su principal objetivo es separar la modificación del DOM de la lógica de la aplicación. Esto ayuda a la reutilización del código y al desarrollo en paralelo. En el patrón MVC los diferentes aspectos de de la aplicación son separados en componentes para separar las responsabilidades. Modelo: Contiene los datos y la lógica. Vista: Disposición visual y la presentación. Controlador: Conecta a la vista con el controlador.

Aplicaciones de una sola pagina

Modelo Tradicional

En el modelo tradicional, todas las transacciones con el servidor se cargaban mediante el recargado completo de la pagina.

SPA

Las principales ventajas son: La aplicación es más fluida y responsiva ya que no tiene que hacer el recargado de la página web en cada llamada. Al mandar la información de la aplicación a través de JSON crea una separación entre la vista (HTML) y lógica de la aplicación.

AngularJS extiende HTML

Nuevos elementos en HTML

En el pasado los diseñadores web no podian crear un código HTML entendible, porque estaban limitados por el diccionario que tenían. Incluso con HTML5 las restricciones continuan. AngularJS permite la compatibilidad de etiquetas HTML entre navegadores sin la necesidad de depender de HTML5.

Componentes Web

Los componentes web consisten de diferentes tecnologías separadas. Puedes pensar en los componentes web como piezas de interfaces de usuario reutilizables que son creados con tecnología web libre. Son parte del navegador y no necesitan librerías externas simplemente importando el componente en el HTML.

Facil conexion entre vista y controlador

Con AngularJS solo se necesita unas cuantas líneas de código para la conexión.

Injeccion de dependencias

La inyección de dependencia es un patrón de diseño de software en el cual a un objeto se le es dado su dependencia, en vez de que el objeto la cree. Ayuda a la modularización de la aplicación. Ayuda al desarrollo independiente entre la dependencia y el dependiente mientras éstos mantengan un contrato conocido. Ayuda a la aplicación a ser más fácil para las pruebas.
